3.6.2 Creación de Objetos con Python

Ahora veamos como definir la clase Alumno con Python.

En python para definir una clase se ocupa la palabra reservada class una clase es un segmento de código por lo cual se encuentra delimitada por su correspondiente identación, veamos como quedaría esta clase con solamente los atributos:

class Alumno:
	matricula
	nombre

¿Modificadores de acceso en Python?

En el lenguaje python no existen modificadores de acceso ya que en python no existen atributos o métodos privados, sin embargo en la documentación oficial podemos encontrar una alternativa basada en la convención del doble guión bajo (_ _).

Dentro de una clase podemos agregar métodos los cuales son segmentos de código por lo cual van delimitados por sus separadores({}) veamos como quedaría la clase Alumno al agregar los métodos:

class Alumno:
	def __init(self)__:
		self.matricula = 'it20150012'
		self.__nombre= 'default Nombre'

3.6.2.1 Prueba de objetos con Python

Notar que tenemos dos atributos el primer atributo matricula y el segundo __nombre, veamos el comportamiento comportamiento del siguiente código

juan = Alumno()
''' esto es correcto y lo mas común'''
print("Matricula: ", juan.matricula, "\n\n")

''' Esto genera un error del tipo: AttributeError'''
print(juan.__nombre)
// Esto genera error por que no existe la variable __nombre

Esto nos produce la siguiente salida:

Matricula: it20150012

Traceback (most recent call last):
  File "codigos/alumno.py", line 6, in <module>
    print(juan.__nombre)
AttributeError: 'Alumno' object has no attribute '__nombre'

Como podemos ver no existe ningún problema con acceder a juan.matricula, sin embargo al acceder a la linea print(juan.__nombre) se genero un error del tipo AttributeError indicando que Objeto del tipo Alumno no tiene el atributo __nombre.

¿Que es lo que pasa?

En realidad lo que pasa es que Python la variable __nombre deja de ser accesible desde el exterior con ese nombre ya que existe un mecanismo denominado name mangling el cual “convierte” a dicha variable como _Alumno__nombre para el acceso desde el exterior, como podemos ver en el siguiente código:

class Alumno:
	def __init__(self):
		self.matricula = "it20150012"
		self.__nombre= "default Nombre"
juan = Alumno()
# Accedemos para imprimir el nombre
print(juan._Alumno__nombre, "\n\n")
# Modificando el nombre
juan._Alumno__nombre = "Juan"
# Accedemos para imprimir el nombre
print(juan._Alumno__nombre, "\n\n")

El cual nos genera la siguiente salida:

default Nombre

Juan

Usted puede descargar el código de ejemplo para ver este comportamiento:

alumno.py
