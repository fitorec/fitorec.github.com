3.1 Definición y elementos de una ontología.

Ontología:

Una Ontología es una definición formal de tipos, propiedades, y relaciones entre entidades que realmente o fundamentalmente existen para un dominio de discusión en particular.

Es una aplicación práctica de la ontología filosófica, con una taxonomía.

Areas de aplicación

Una ontología cataloga las variables requeridas para algún conjunto de computación y establece las relaciones entre ellos:

  • En los campos de la inteligencia artificial
  • La Web Semántica.
  • Ingeniería de sistemas
  • Ingeniería de software
  • Informática biomédica
  • Bibliotecología
  • Arquitectura de la información se crean ontologías para limitar la complejidad y para organizar la información.

Vamos a explicar el concepto de odontología a partir de la Web Semántica.

Que es La Web Semántica?

Una Web extendida, dotada de mayor significado en la que cualquier usuario en Internet podrá encontrar respuestas a sus preguntas de forma más rápida y sencilla gracias a una información mejor definida.

La Web Semántica E Internet de las cosas:

Gracias a la semántica en la Web, “las cosas”(basadas en el software) son capaz de procesar su contenido, razonar con este, combinarlo y realizar deducciones lógicas para resolver problemas cotidianos automáticamente.

Las ontologías nos sirven para definir la semántica a los contenidos permitiendo compartir esta información entre “cosas conectadas a Internet” esto es denominado Internet de las Cosas(*IoT*Internet of Things).

3.3 Lenguajes de representación.

Es necesario usar un lenguaje que permita definir ontologías para ser usado en aplicaciones que necesitan procesar el contenido de la información por software en lugar de únicamente representar información para los humanos.

Para esto el W3C define el El Lenguaje de Ontologías Web (OWL Ontology Web Language), el cual es un lenguaje que facilita el mecanismo de interoperatividad.

Mayor Información: https://www.w3.org/2007/09/OWL-Overview-es.html

Ontología en la web.

Las etiquetas HTML le indican al navegador cómo mostrar la información incluida en las etiquetas. Por ejemplo <h1>Avatar</h1> indica al navegador que muestre la cadena de texto "Avatar" en el formato de título 1.

Sin embargo, no da ninguna información sobre qué significa esa cadena de texto: "Avatar", lo que dificulta la búsqueda en motores para mostrar el contenido relevante.

Mediante la adición de atributos adicionales a las etiquetas al marcado HTML, podemos ayudar a los motores de búsqueda y otras aplicaciones a comprender mejor tu contenido.

Documetación oficial: Getting Started - schema.org

Una solución es hacer uso de la API de Microdatos de schema.org, expliquemos esto con un ejemplo.

Ejemplo uso de la api Schema.org

Imagina que un amigo te solicita ayuda con su sitio web de descargas de películas, dice que su mayor problema es que no tiene visitas, al visitar su sitio web en donde tiene la película de Avatar, su código HTML genera esto:

<div>
 <h1>Avatar</h1>
 <span>Director: James Cameron (agosto 16, 1954)</span>
 <span>Ciencia Ficción</span>
 <a href="../trailers/avatar-theatrical.html">Trailer</a>
</div>

Si probamos el código en el tester https://search.google.com/structured-data/testing-tool de google podremos que no el html no contiene información semantica(cadente de sentido).

<div itemscope itemtype ="http://schema.org/Movie">
  <h1 itemprop="name">Avatar</h1>
  <div itemprop="director" itemscope itemtype="http://schema.org/Person">
	Director: <span itemprop="name">James Cameron</span>
	(<span itemprop="birthDate">agosto 16, 1954</span>)
  </div><!-- fin Person-->
  <span itemprop="genre">Ciencia Ficción</span>
  <a href="../trailers/avatar-theatrical.html" itemprop="trailer">Trailer</a>
</div><!-- fin Movie-->

Si probamos el nuevo código HTML con la misma herramienta, veremos que ya nos aparece la información interpretada, ¿pero que hicimos?, vamos a comentar ciertas cosas:

Definición de elementos(itemscope y itemtype)

itemscope(Item Scope) este atributo indica que esta etiqueta va a contener una “identidad de algo”(clase) por lo cual se suele acompañar del atributo itemtype(Item Type) el cual contiene la URL de la referencia de esa cosa, en este ejemplo podemos ver que existen dos etiquetas con estas caracteristicas.

<div itemscope itemtype ="http://schema.org/Movie">
	<div itemprop="director" itemscope itemtype="http://schema.org/Person">
	</div><!-- fin Person-->
</div><!-- fin Movie-->

Como podemos ver podemos anidar cosas, es decir en este caso un tipo Pelicula(Movie), uno de sus atributos(itemprop="director") es del tipo Persona(Person).

Definición de atributos(itemprop)

itemprop(Item Property) nos permite definir propiedades o atributos para los Scopes, esta suele tener la siguiente sintaxis:

<tag itemprop="nombre-propiedad">Valor de la propiead</tag>

Por ejemplo el elemento Movie contiene una propiedad(itemprop) con nombre name con valor Avatar, como podemos ver en el siguiente código:

<div itemscope itemtype ="http://schema.org/Movie">
	<h1 itemprop="name">Avatar</h1>
</div><!-- fin Movie-->
